# M2 LOG Tool

實驗 LOG 收集與輸出工具。左邊填寫實驗欄位，右邊貼上 **UEFI** / **SAM** LOG，按下「輸出」即自動建立目錄並寫入對應檔案，並可一鍵開啟檔案總管。

## 功能

- 左側：實驗欄位（名稱、日期、測試人員、測試項目、備註）+ 可自行新增 / 移除的自訂欄位
- 右側：UEFI / SAM 分頁的大型 LOG 貼上區，含行數 / 字元計數
- **輸出**：自動建立 `<輸出根目錄>/<YYYYMMDD>_<HHmm>_<10碼英文縮寫>/`，內含：
  - `info.json`：實驗欄位（含自訂欄位）
  - `UEFI/UEFI.log`（有內容才產生，實驗資訊崁在 LOG 最上面）
  - `SAM/SAM.log`（有內容才產生，實驗資訊崁在 LOG 最上面）
  - `<資料夾名稱>.zip`：同目錄下的完整壓縮檔
- 實驗名稱**必須是英文**，並自動縮寫成 10 碼當資料夾名稱
- 左右面板中間可**拖曳調整寬度**（往右拉放大左側、右側自動縮小，反之亦然）
- **檔案總管**：用系統檔案管理員開啟最近輸出的資料夾
- 輸出後表單與自訂欄位**不會被清空**（localStorage 自動記憶）、`Ctrl + Enter` 快速輸出

## 啟動

```powershell
npm install
npm start
```

開啟瀏覽器：<http://127.0.0.1:3000>

## 輸出結構範例

實驗名稱 `Boot Stress Test` → 縮寫 `BSTOOTTRES`：

```
LOG_OUTPUT/
└─ 20260611_1430_BSTOOTTRES/
   ├─ info.json
   ├─ 20260611_1430_BSTOOTTRES.zip   ← 同目錄內的完整壓縮檔
   ├─ UEFI/
   │  └─ UEFI.log   ← 最上面崁入實驗資訊
   └─ SAM/
      └─ SAM.log    ← 最上面崁入實驗資訊
```

## 設定

- 預設輸出到專案內 `LOG_OUTPUT/`。可在「輸出根目錄」欄位填絕對路徑（例如 `D:\Logs`）。
- 連接埠：環境變數 `PORT`（預設 `3000`）。
- 伺服器只綁定 `127.0.0.1`，不對外網路開放。
